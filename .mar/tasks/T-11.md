Two related improvements to `mar serve`:

1. Multiple instances: serve binds a fixed 127.0.0.1:7777, so a second `mar
   serve` in another project fails with "address already in use". Make the
   default port auto-fall-back to a free port when the requested one is taken
   (try the default, then bind :0 and report the chosen port), so several
   projects can be browsed at once. An explicit --port stays authoritative
   (error if that exact port is busy).

2. Show the project path on every page so it's clear which project/store you're
   looking at when multiple are open. Pass the absolute store/project path into
   the layout and render it in a small header on index, board, doc, and task
   pages.
