---
title: seed() re-seeds default columns whenever columns table is empty
status: archived
created: "2026-06-09T17:45:08.858811Z"
updated: "2026-06-10T21:19:05.278667Z"
---
Found while reproducing readcube/go-store's board in mar. Repro: remove all
columns (e.g. to rebuild them in a custom order), then run any mar command. The
next Open() calls seed(), which checks `SELECT count(*) FROM columns` and, on
zero, re-inserts the default To do / In progress / Done. So a deliberately
emptied board silently repopulates with defaults, and subsequent `column add`
calls then collide on the UNIQUE name constraint.

Root cause: seed() keys "is this a fresh DB?" off an empty columns table rather
off whether the store was just created. Fix: only seed on Init() (first
creation), not on every Open(); or gate seeding on a persisted meta flag
(e.g. meta.seeded=1) set once at init. This also affects anyone who wants a
board with a custom column set that doesn't include the three defaults.
