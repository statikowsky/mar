Currently each board card (`internal/web/templates/board.gohtml`) is an
`<a href="/task/{code}">`, so clicking it navigates away to the standalone task
page (`handleTask` / `task.gohtml`). Requested behaviour: clicking a card opens
the task inline as a popup/modal overlay on the board, without leaving the page.

Sketch:
- Add a modal container + lightweight JS to `board.gohtml` (or the layout).
- On card click, fetch the task content and show it in the overlay instead of
  following the link. Two options: (a) fetch the rendered `/task/{code}` page and
  extract its content, or (b) add a `/task/{code}?fragment=1` (or a small JSON/
  partial endpoint) that returns just the task body + linked docs for the modal.
- Keep the link as a graceful fallback (progressive enhancement): no-JS still
  navigates to the full page; Escape / click-outside closes the modal; update
  the URL hash so a card is deep-linkable.

Found while reviewing the go-store board reproduced in mar.
