Today the web board (internal/web/templates/board.gohtml, handleBoard) is
read-only: cards render in column/position order but can only be moved via the
CLI (`mar task move`). Add browser-side reordering so the user can drag a card
within a column or across columns, persisting the new column + position.

Sketch:
- Make cards draggable (HTML5 drag-and-drop or a small dependency-free pointer
  handler) in board.gohtml; columns are drop targets.
- On drop, POST the move to a new write endpoint (the web layer is currently
  read-only) — e.g. POST /task/{code}/move {column, after} — that calls the
  existing store.MoveTask. This is the first server-side mutation route; keep it
  loopback-only like the rest of serve.
- Reuse the gap-based positionAfter logic; the "after" card is whatever the drop
  position lands between.
- SSE already broadcasts on data_version change, so other open tabs reload.
- Progressive enhancement: no-JS still shows the ordered read-only board.

Found while reviewing the reproduced go-store board; complements T-6 (open card
in a modal).
