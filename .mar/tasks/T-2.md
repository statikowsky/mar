go-store's kanban uses meaningful, non-sequential task codes (T-01, T-39, T-41)
with gaps and zero-padding. mar auto-numbers T-1, T-2... with no override, so
the original codes could not be preserved when recreating the board. Add an
optional `--code` on `task create` (validated/unique, like doc codes), keeping
auto-numbering as the default.
